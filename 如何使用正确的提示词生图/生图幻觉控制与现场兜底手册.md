# 生图幻觉控制与现场兜底手册

> 配套《如何使用正确的提示词生图》教案 · 现场速查
> **核心原则：翻车不是事故，是教学时刻；无论如何，让孩子有作品带走。**

---

## 一、先分清"翻车"的两类（应对方式不同）

1. **内容翻车（AI 幻觉）**：多指 / 多肢、违反物理、文字乱码、角色漂移、画风崩坏、严重不符合意图。
2. **系统翻车**：API 失败 / 限流、断网、内容审核拒绝、生成太慢、账号额度用尽。

---

## 二、控制 / 减少幻觉（预防，课前 + 提示词层）

### 提示词层
- **数量与特征写清**："一只橘猫"而非"猫"；主体特征明确，减少 AI 自由发挥。
- **锁绘本 / 插画风格**：比写实容错高、对儿童更安全；**写实人脸和手部特写最易翻车**，尽量回避，或让手被遮挡。
- **不要求 AI 画文字**：文字一律后期加（项目 config 已用 `burn_subtitles` 这么处理）。
- **避免矛盾指令**；控制场景复杂度，元素越少越可控。
- **负面提示词**（工具支持时）：排除 `extra fingers / deformed / text / watermark / blurry`。

### 工具设置层
- **优先"图生图"而非纯文生图**：给参考图锁构图与角色——这是控制角色漂移最有效的手段。（同事的「儿童 AI 故事视频」MVP 正是用"原画 + 角色标准图"作参考来锁角色一致性，思路一致。）
- **best-of-n**：一次生 3–4 张选最好的，翻车概率大幅下降。
- **锁 seed** 保持一致；适当降低随机性参数。
- 用 3:4 竖版（项目已定）。

### 内容安全层
- 预设安全风格（童趣 / 绘本），避开写实儿童人脸。
- 开内容审核；教师巡场。

### 课前预演（最重要）
- **教师先按教案跑通样片**（项目 SOP 铁律：先自己跑出 1 支样片再带给孩子）。
- **预生成一批"保底图"存档**（见第四节），现场可随时调用。

---

## 三、现场翻车应急（分级响应）

### 3.1 内容翻车（多指 / 乱码 / 漂移）
- **第一反应：别删！** 变成"看图挑错"教学素材——"看，AI 又猜错了，一起找找它哪儿错。"（正好接教案的"看图也要挑错"环节）
- **技术救**：改一句话再生成（当场演示迭代 = 教学）；换图生图传参考图；换绘本风格；best-of-n 选张好的。

### 3.2 出敏感 / 不当内容
- **立即切换 / 关掉**，不展开、低调处理，阻止孩子截图传播。
- 课后复盘提示词，避开触发词。

### 3.3 系统翻车（API / 限流 / 断网 / 审核拒）
- **切备用工具 / 账号**（豆包 + 即梦双备）。
- **切"预生成保底图库"**（见第四节）。
- **审核拒绝**：改提示词措辞（去掉敏感词），换主题。

---

## 四、兜底（无论如何有作品带走）

1. **预生成保底图库**：课前按每个"灵感图卡" / 常见主题预生成合格图存本地。现场 AI 不可用时，让孩子做"挑图 → 看图反推提示词 → 改词二次创作"，仍有完整产出。
2. **双工具双账号**：主备各一，一个挂了立刻切。
3. **离线退路**：全场断网 → "纸上写提示词，老师课后帮生成发群里"；首映改用"老师课前样片 + 孩子写的提示词"，闭环不断。
4. **半成品兜底**：哪怕只生出一幕 / 一张，也围绕它做"挑错 + 改词"，照样有作品感。
5. **时间兜底**：翻车超时 → 跳过"三连图"，保住单图 + 家长首映。

---

## 五、翻车响应速查表（现场贴在讲台）

| 现象 | 立即动作 | 教学说辞 |
|---|---|---|
| 多指 / 乱码 / 角色漂移 | 不删→当挑错素材→改词或换图生图重生 | "AI 猜错了，咱们一起找 bug" |
| 不符合意图 | 问孩子"哪句没说清"→改词重生 | "不是 AI 笨，是没说清" |
| 敏感 / 不当内容 | 立即切换 / 关掉，低调处理 | 不展开，换主题 |
| API 失败 / 限流 | 切备用工具 / 账号 | "工具忙，换一个" |
| 断网 | 切预生成图库 / 纸上写词课后补 | "咱们先把话写好" |
| 翻车超时 | 跳三连图，保单图 + 首映 | "时间紧，先保作品" |

---

## 六、一句话原则

> **预防靠"参考图 + 绘本风 + best-of-n"；现场靠"翻车即教学"；兜底靠"预生成图库 + 双工具"。无论如何，孩子要有作品带走。**

# 程序员团队授课指南 · 试讲内测优先

> 前提更新：本课程目的 = 线下免费培训**积累教学经验**，非推销。团队均为程序员，**无授课经验**。
> 由此：成功标准从"转化率"改为"教学流程跑通 + 经验沉淀"；最大风险从"护城河"转为"**讲不好 / 控不住场**"。
> 你们的强项是技术链路（跑通生图、预生成、账号额度）——这块不用教；下面专治"教学交付"这个短板。

---

## 一、重新定义"这场算赢"（别用转化衡量）

不看转化、不看付费。这场是**教研内测**，看 4 件事：
1. **流程是否跑通**——时间、设备、翻车兜底是否如设计。
2. **孩子是否投入**——注意力曲线、参与度（不是"开心"，是"在跟"）。
3. **哪里崩了**——记录每个崩点，下次改。
4. **经验沉淀**——带出 3 条可复用的教训。

> 所以：把"护城河话术 / 转化软接 / 能力证据卡"从必做降为**可选**；把"试讲内测 + 复盘"升为**必做**。

---

## 二、程序员教小孩的 6 个典型坑（+ 对策）

| 坑 | 程序员惯性 | 对策 |
|---|---|---|
| 1. 讲原理上瘾 | 爱解释"AI 怎么训练 / 参数 / 模型" | 原理 ≤ 1 句话（教案已有），其余全"演示+动手"；孩子不在乎底层 |
| 2. 用成人节奏 | 一段讲 15 分钟不停 | 单段 ≤ 8 分钟，"讲→做→看"高频切换；生图等待必插纸上任务 |
| 3. 术语脱口而出 | prompt / seed / 参数 / 模型 | 换孩子话："提示词=给 AI 的任务单""seed=AI 的骰子号"。备一份术语转译表 |
| 4. 控不住场 | 15 个孩子+设备，噪声/走神/卡住 | 课堂口令 + 助教巡场 + 分组组长制 + 设备规则前置 |
| 5. 演示翻车慌 | 现场生图必翻车，易进"调试模式"卡住冷场 | 课前预生成演示结果（不靠现场网络）；翻车即"找 bug 游戏"，不调试、不道歉过度 |
| 6. 不读场 | 看不出孩子 lost / bored | 设"信号员"（助教专盯表情/进度）；主讲每 5 分钟扫一次"还有多少人在跟" |

---

## 三、试讲内测流程（强烈建议先内部跑，别直接上 15 人）

1. **内测 1：团队互讲**（无孩子）——1 人当老师讲完整流程，其他人扮"刁钻孩子"（提问/走神/卡住）。目标：卡时间、顺话术、找讲不顺的点。**录屏复盘**。
2. **内测 2：小范围真孩子**（3–5 个亲友/同事孩子）——低风险验证，重点看孩子注意力与翻车处理。
3. **正式首场**：15 人社区场，按内测改进版执行。
4. 每场后 **30 分钟教研复盘**，填下表。

> 程序员别跳过内测 1——你们最缺的是"开口讲"的肌肉记忆，不是再改一版文档。

---

## 四、备课重点转移（你们容易搞反）

- ❌ 别把时间花在"做更炫的 PPT / 再改教案文档"上。
- ✅ 花在"练嘴"上：把**开场、三档对比、找错游戏、首映话术**对着镜子/同事讲 3 遍，卡时间到分钟。
- ✅ 课前用统一账号**自己跑通完整流程一遍**（生图 + 兜底图 + 图文小卡），确保链路通——这是你们的强项，但别漏"账号额度 / WiFi 并发"两个坑。

---

## 五、课堂控场小工具（给非教师，亲测有效）

- **课堂口令**：定一个专注信号，如"3-2-1，屏幕朝下"，孩子响应——比喊"安静"管用。
- **进度可视化**：白板画进度条/打卡表，孩子看到"还剩几步"，减少"什么时候结束"的焦躁。
- **分层任务盒**：提前备"快孩子挑战卡"和"卡住孩子提示卡"，助教按需发，避免干等和无聊。
- **计时器**：每环节倒计时投屏，主讲和孩子都看时间，防拖堂。
- **"信号员"助教**：专盯全场表情和进度，发现集体走神立刻给主讲暗号。

| 孩子投入 | 注意力掉点时刻 | |
| 翻车 | 翻了几次、怎么处理的 | |

## 八、一句话执行原则

> **这场是教研内测，不是招生。程序员别讲原理、别现场调试、别用术语；先团队互讲、再 3-5 个真孩子、再 15 人场；每场后填复盘表沉淀经验。技术你们稳，重点全压在"练嘴+控场+复盘"上。**

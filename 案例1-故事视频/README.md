# 儿童 AI 故事视频

把孩子的一张手绘原画变成一个有配图、有孩子声音、可以发到微信的 3:4 故事视频。

整个过程由 WorkBuddy、TRAE Work 或其他支持 Agent Skills 的工具执行，不需要使用剪映，也不调用动态视频模型。

## 孩子需要做什么

1. 画一个喜欢的角色或物品。
2. 回答几个简单问题，共同决定故事。
3. 录制 15-30 秒清晰声音。
4. 选择喜欢的故事方向和最终配图。

其余步骤由 Skill 自动完成：整理原画、生成故事配图、克隆声音、加入字幕并输出 MP4。

## 文件夹说明

| 文件夹 | 放什么 |
|---|---|
| `input/` | 本次要使用的原画和参考声音（位于工作区根目录） |
| `项目名/` | 每个孩子生成的独立文件夹（生成在工作区根目录） |
| `脚本/` | 自动生图、配音、合成和检查程序（含单元测试） |
| `参考/` | 视频创作流程、故事规则、技术和隐私说明 |
| `样例/` | 不含真实儿童信息的示例故事 |
| `资源/` | 可选的片头、字体和其他公共素材 |
| `评估/` | 用于检查 Skill 是否容易正确触发的测试任务 |

第一次使用，只需要阅读本文件。Agent 会根据任务自动读取其他说明。

## 第一次准备

1. 确保电脑已安装 Python 3.10 或更高版本，以及 FFmpeg。
2. 复制 `.env.example` 为 `.env`。
3. 在 `.env` 中填写 `VOLCENGINE_API_KEY` 和 `MIMO_API_KEY`。
4. 不要把 `.env` 发给其他人，也不要把密钥直接写进故事文件。

运行环境检查：

```bash
python3 脚本/run.py preflight
```

## 制作方式（三阶段逐步执行与停顿核对）

通常不需要手动输入命令，Agent 会根据 Skill 的定义逐步执行。若需要手动在终端中运行，请遵循以下三阶段以确保在关键步骤有停顿核对的机会：

### 阶段 1. 根据要求生成文稿
运行初始化命令生成故事文稿和初始版 Word 文档：
```bash
python3 脚本/run.py init \
  --project "项目名" \
  --drawing "input/原画.jpg" \
  --story "参考/story.example.json" \
  --voice "input/声音.m4a"
```
* **【停顿核对 1】**：请打开项目根目录下的 `故事.docx`，检查文案和设定是否符合预期。如需修改，请直接编辑并保存 Word 文档。

### 阶段 2. 生成配图与声音旁白
文稿核对无误后，运行以下命令生成图片并克隆/合成声音：
```bash
# 生成四幕配图并自动插入到 故事.docx 中
python3 脚本/run.py images --project "项目名"

# 生成各幕旁白音频
python3 脚本/run.py voice --project "项目名"
```
* **【停顿核对 2】**：重新打开 `故事.docx` 检查插入的图片和故事排版是否满意，并到项目的 `声音/` 目录下试听生成的音频。如需修改，可直接编辑 Word 并再次运行该命令。

### 阶段 3. 合成视频
核对图片和声音无误后，合成最终的故事视频并进行兼容性校验：
```bash
python3 脚本/run.py render --project "项目名"
python3 脚本/run.py verify --project "项目名"
```

## 最终产物

执行完所有步骤后，项目目录下会生成并保留以下文件：

- `故事视频.mp4`：最终导出的 1080×1440 视频文件（含字幕与配乐，可直接分享）。
- `故事.docx`：核心故事文档（包含角色设定、旁白文本、提示词以及插入好的高清故事配图）。
- `.工作/故事.json`：保存整个故事配置结构的数据文件。
- `图片/`：存放生成好的角色标准参考图和四幕高清配图的文件夹。
- `声音/`：存放各幕生成好的旁白音频文件夹。
- `.工作/校验结果.json`：自动化兼容性与质量校验报告。

## 未成年人声音

声音克隆会把参考录音发送给小米 MiMo 服务。正式课堂使用前，应获得监护人明确同意。默认情况下，Skill 在视频成功后删除为了上传而生成的标准化声音副本；原始录音是否保留由老师自行决定。

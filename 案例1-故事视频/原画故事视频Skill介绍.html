02 / 制作流程

文本起步，四阶段、三次确认。

先确认文稿，再确认角色原型；前三步都停下来检查，避免角色走样后整批返工。

STAGE 01

讲故事，出文稿

先说清角色、想去哪里、遇到什么困难和希望的气氛。Skill 生成初版 故事.docx，此时不需要原画。

确认点 1：打开 Word，直接修改并保存。
STAGE 02

加入原画，定原型

文稿确认后再提供原画。此时只生成角色原型，不生成场景图或旁白。

确认点 2：确认角色外形、颜色和手绘气质。
STAGE 03

出场景，听旁白

只使用已确认的原型生成四幕配图和旁白；图片会自动写回 Word。

确认点 3：检查配图与音频，再决定是否合成。
STAGE 04

合成并交付

将图片、声音、字幕、BGM 和结尾署名页合成 1080×1440 MP4，并执行完整性校验。

交付：分享前检查不希望公开的信息。
03 / 直接启动

复制一段话，就能从故事开始。

不需要输入命令。把下面的示例作为第一条消息发给 Agent；它会先与你共创故事，再只生成 Word 文稿并停下来等待确认。

  1. 先说故事
    只需给出角色、发生的事和你想要的感觉。原画和声音可以在文稿确认后再准备。
  2. 确认 Word
    在 Word 中修改旁白、画面提示词、配音音色和背景音乐；确认后再提交原画。
  3. 确认原型
    Agent 只生成角色原型。先检查角色是否像你想象的那样，再允许生成四幕场景。
  4. 继续制作
    Agent 再生成图片、声音与视频；不满意时，用自然语言说“重做第二幕”即可。
复制后，直接粘贴给 Agent
请帮我用「原画故事视频」Skill，为小学六年级学生创作一个可微信分享的竖版故事视频。

故事想法：诸葛亮接到寻找失落军令的任务，意外穿越到今天的校园。他要在放学前找到线索，回到三国。

风格：轻松、有智慧、不说教。

请先给我三个一句话故事方向。选定后，只生成可编辑的 故事.docx 文稿，并停下来等我确认；现在不要生成图片、声音或视频。

文稿确认后先补充一句：“这是我的原画，请只生成角色原型并停下来等我确认。”
原型确认后再说：“请使用已确认的角色原型生成四幕配图和旁白。”

04 / 关键约定

创作方便，也要可控。

这几条规则能让项目安全、可复用，并让最终成片真正反映 Word 中的最新修改。

编辑与重做

  • 故事.docx 是核心编辑入口；保留旁白和提示词的开始/结束标记。
  • 原型不满意时，先修改角色设定并重做原型；未确认前不要生成四幕场景。
  • 只重做一幕时，直接告诉 Agent “重做第 N 幕”；其他素材不会重复生成。

素材与隐私

在第二阶段上传原画前确认拥有使用权；上传声音前获得录音者明确同意。不要把完整姓名、住址、学校、联系方式写入项目名、故事或提示词。

# 儿童 AI 故事视频

## 目标

让孩子保有创作主导权，同时把技术步骤压缩为可理解的选择。最终交付一个 1080×1440、H.264/AAC 编码、带旁白和字幕的 MP4，不要求用户使用传统剪辑软件。

## 开始前

1. 阅读根目录 `README.md`，理解目录和最简使用方式。
2. 运行 `python3 脚本/run.py preflight`。缺少环境或密钥时，说明缺少什么以及如何补齐，不要展示密钥内容。
3. 确认已取得监护人对声音克隆的明确同意。未确认时，可以继续创作故事和生成图片，但暂停声音上传。
4. 不把孩子真实姓名、学校、住址或其他身份信息发送给外部模型。

## 和孩子一起确定故事

保持问题短、一次只问一组。优先从用户已有信息中提取答案，缺少时询问以下五项：

1. 画的是什么，它叫什么名字？
2. 它最特别的地方或超能力是什么？
3. 它想去哪里？
4. 它遇到了什么小困难？
5. 你希望故事是开心、勇敢、温暖还是搞笑？

根据回答给出三个一句话故事方向，让孩子选择。选择后按 `参考/02_故事创作规则.md` 写成四幕故事，并保存为项目的 `文本/故事.json`。字段结构参考 `参考/story.example.json`。

从原画推断 `character.subject_type`。当主角是动物、物品、植物、车辆或建筑时，故事的行动者就是它本身，不要擅自改成人类小孩或给它增加人类身体。

故事确认是第一个停顿点。不要在孩子还没选定方向时直接生成全部素材。

## 制作流程

### 必须遵守的【三节点停顿确认】规则：
1. **阶段 1：根据要求生成文稿**
   - 运行项目初始化和生成 DOCX 脚本。
   - **【停顿点 1】**：必须在此处停止，明确告知用户打开项目根目录下的 `故事.docx`，对角色设定、旁白和画面提示词进行检查和修改。**在用户确认文稿无误前，严禁运行第二阶段命令。**
2. **阶段 2：根据文稿生成图片和音频**
   - 运行图片生成和音频生成脚本。生成的图片将自动插入到 `故事.docx` 中。
   - **【停顿点 2】**：必须在此处停止，明确告知用户重新打开 `故事.docx`，检查生成的配图与旁白是否符合预期，并听听音频。**在用户确认图片和音频无误前，严禁运行第三阶段命令。**
3. **阶段 3：合成视频**
   - 运行视频合成和校验脚本，渲染最终的 3:4 MP4 故事视频。

---

### 阶段 1. 根据要求生成文稿

与孩子通过聊天交互生成包含明确分段和格式的故事（可参考 `参考/story.example.json` 结构），并保存为项目的 `.工作/故事.json`。

保存故事 JSON 后，运行初始化命令生成初始版 `故事.docx`（各幕之间有明显的符号隔开，包含图片占位符）：

```bash
python3 脚本/run.py init \
  --project "项目名" \
  --drawing "原画路径" \
  --story "故事JSON路径" \
  --voice "声音路径"
```

项目名使用昵称或编号，不使用孩子完整姓名。

> [!IMPORTANT]
> 项目的核心文件是根目录下的 `故事.docx`。用户和孩子可以直接在 `故事.docx` 中对角色设定、画面风格、旁白（`【旁白-开始/结束】`）和画面提示词（`【画面提示词-开始/结束】`）进行人工修改。在后续生成图片、音频和视频的步骤中，脚本会自动先从 `故事.docx` 中解析出最新内容并同步覆盖 `.工作/故事.json`，确保最终生成的配图和视频反映最新的修改。
> 
> **在完成阶段 1 后，Agent 必须停下来，提供 `故事.docx` 的绝对路径，并提示用户：“故事文稿已生成并初始化为 Word 文档。请打开该文档核对故事内容、角色特征以及画面提示词。如果您做了任何修改，直接保存 Word 文件即可。确认无误后请回复，我将为您生成图片和音频。”**

---

### 阶段 2. 根据文稿生成图片和音频

在用户/孩子确认文案无误后，依次运行以下两个步骤（生成图片和音频）：

1. **生成故事配图并自动插入 DOCX**：
   ```bash
   python3 脚本/run.py images --project "项目名"
   ```
   脚本会先生成角色标准图，再将“孩子原画＋角色标准图”同时作为参考生成四幕3:4配图。默认模型和尺寸来自 `config.json`。生成的配图会自动同步插入到项目的 `故事.docx` 文档对应段落中。
   
   若需单独重做某一幕图片：
   ```bash
   python3 脚本/run.py images --project "项目名" --scene 2 --force
   ```

2. **生成声音旁白**：
   ```bash
   python3 脚本/run.py voice --project "项目名"
   ```
   每幕单独生成 WAV 音频，便于准确对齐图片和字幕。

   **声音生成逻辑（二级模式）：**
   * **第一级（个性化克隆，模型 `mimo-v2.5-tts-voiceclone`）**：
     * **触发条件**：项目 `输入/` 下存在 `声音参考.*` 文件（支持 `.wav`、`.mp3`、`.m4a`）。
     * > [!WARNING]
     * > **禁止**将官方预置声音样品（如 `白桦_成年男.wav` 等）复制到 `输入/` 下作为 `声音参考.wav`！这会导致对官方音色做不必要的二次克隆，既降低合成质量又浪费额度。
   * **第二级（官方预置音色，模型 `mimo-v2.5-tts`）**：
     * **触发条件**：项目 `输入/` 下没有提供任何个性化声音参考。
     * **音色选择优先级**：命令行 `--preset` 指定 > `故事.docx` 文档中指定的 `★ 配音音色` > 全局配置文件 `config.json` 中的 `preset_voice`（默认值）。
     * **可选官方音色 ID**：
       - `冰糖`（女孩 / 甜美）
       - `苏打`（男孩 / 活力）
       - `茉莉`（成年女 / 温柔）
       - `白桦`（成年男 / 磁性）

> [!IMPORTANT]
> **在完成阶段 2 后，Agent 必须再次停下来。提示用户：“配图和音频旁白均已生成，图片已同步插入到 Word 文档中。请打开 `故事.docx` 查看排版效果，并前往音频目录核对旁白音频。如果有任何不满意，可以直接在 Word 文档中修改提示词或旁白文本。确认无误后请回复，我将为您合成最终的视频。”**

---

### 阶段 3. 合成视频

在用户确认配图和音频没有问题后，运行以下命令合成最终视频，并进行兼容性校验：

```bash
python3 脚本/run.py render --project "项目名"
python3 脚本/run.py verify --project "项目名"
```

最终视频 `故事视频.mp4` 保存在项目的根目录中。可以通过 `.工作/校验结果.json` 检查生成是否成功。

> [!CAUTION]
> **为严格保证每个阶段用户有充分的检查机会，不要使用 `run.py all` 命令一键执行所有流程。请遵循上述三个阶段逐步推进并执行停顿确认。**

## 失败处理

- 外部接口限流或临时失败：按脚本内置退避重试，保留已完成素材。
- 某一幕角色漂移：只做该幕，并继续同时传入原画与角色标准图。
- 参考录音噪声太大：请重新录制15-30秒、单人、近距离、无背景音乐的声音。
- 字幕烧录不可用：先保留 SRT 和无烧录字幕版视频，不阻塞作品交付。
- 网络中断：再次执行原命令，脚本会复用已有文件。

## 交付时说明

用孩子和家长容易理解的语言说明：孩子决定了角色和故事，AI帮助扩展配图和合成，最终视频使用了孩子授权的声音。提供最终 MP4 的路径，并提醒家长分享前检查是否包含不希望公开的信息。

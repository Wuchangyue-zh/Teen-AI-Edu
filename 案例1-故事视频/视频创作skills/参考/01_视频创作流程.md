# 视频创作执行流程

## 核心制作步骤

视频创作遵循以下三个核心阶段，并在每个阶段结束时停下来由用户/监护人核对确认：

1. **第一阶段：根据要求生成文稿并导出 DOCX**
   - **准备与检查**：运行 `python3 脚本/run.py preflight` 验证环境与 API Key 配置。
   - **建立项目**：使用独立编号或昵称建立项目，导入原画和参考声音。
   - **故事创作与初始化**：编写符合规则的四幕故事，保存为项目的 `故事.json`，并初始化生成项目根目录下的 `故事.docx`。
   - **【停顿点 1】**：停下来由用户/监护人核对 `故事.docx` 中的文案、角色设定与画面提示词，支持用户直接在 Word 文档中修改保存。
   
2. **第二阶段：根据文稿生成图片与音频**
   - **图片生成**：运行图片生成命令，基于原画和生成的角色标准图，自动渲染四幕 3:4 故事配图并将其插入到 `故事.docx` 中。
   - **音频生成**：运行旁白声音生成命令，可克隆参考声音或使用官方预置音色生成每幕音频。
   - **【停顿点 2】**：停下来由用户/监护人核对 `故事.docx` 中新插入的画面效果以及试听生成的旁白音频。支持用户直接在 Word 文档中修改并重新生成。

3. **第三阶段：合成视频**
   - **视频渲染与校验**：在图片和音频都核对无误后，运行合成命令，将画作、配乐、旁白和字幕融合为 3:4 的故事视频，并完成质量与隐私校验。

## 制作要点

- **项目隔离**：每个视频创作项目使用唯一的编号或昵称建立，避免文件或配置重名覆盖。
- **阶段性检查**：严格执行三节点停顿确认规则，禁止连续运行下一步生成，把内容的主导权交还给创作者（孩子与家长）。
- **隐私与安全**：禁止在项目名称、故事文本、提示词等处包含全名、学校、住址等真实隐私信息。首发前必须由监护人完成最终内容核对。

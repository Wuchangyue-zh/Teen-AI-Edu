# 原画故事视频执行流程

## 核心制作步骤

视频创作遵循以下四个核心阶段，并在前三阶段结束时停下来由创作者核对确认：

1. **第一阶段：根据要求生成文稿并导出 DOCX**
   - **准备与检查**：运行 `python3 脚本/run.py preflight` 验证环境与 API Key 配置。
   - **故事创作与初始化**：从创作者的一段故事想法出发，编写符合规则的四幕故事，保存为项目的 `.工作/故事.json`，并初始化生成项目根目录下的 `故事.docx`。此时不需要原画或参考声音。
   - **【停顿点 1】**：停下来由创作者核对 `故事.docx` 中的文案、角色设定、画面提示词与背景音乐选项，支持直接在 Word 文档中修改保存。
   
2. **第二阶段：生成角色原型**
   - **补充原画**：文稿确认后，再导入原画。
   - **角色原型**：只生成角色标准参考图并自动插入 `故事.docx`，不生成任何场景图片或音频。
   - **【停顿点 2】**：停下来确认角色原型的外形、颜色、结构与手绘气质。原型未确认前，禁止生成四幕场景。

3. **第三阶段：根据已确认原型生成图片与音频**
   - **图片生成**：基于原画和已确认的角色原型，自动渲染四幕 3:4 故事配图并将其插入到 `故事.docx` 中。
   - **音频生成**：运行旁白声音生成命令，可克隆参考声音或使用官方预置音色生成每幕音频。
   - **【停顿点 3】**：停下来核对 `故事.docx` 中新插入的画面效果以及试听生成的旁白音频。支持直接在 Word 文档中修改；重做单幕图片后 Word 会自动刷新对应配图。

4. **第四阶段：合成视频**
   - **视频渲染与校验**：在图片和音频都核对无误后，运行合成命令，将画作、配乐、旁白和字幕融合为 3:4 的故事视频，并完成质量与隐私校验。

## 制作要点

- **项目隔离**：每个视频创作项目使用唯一的编号或昵称建立，避免文件或配置重名覆盖。
- **阶段性检查**：严格执行四阶段、三次人工确认，禁止连续运行下一步生成，把内容主导权交还给创作者。
- **隐私与安全**：禁止在项目名称、故事文本、提示词等处包含全名、学校、住址等真实隐私信息。分享前由创作者完成最终内容核对。

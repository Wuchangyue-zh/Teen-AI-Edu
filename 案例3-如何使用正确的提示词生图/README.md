# 案例3：如何使用正确的提示词生图

这是一套面向小学 4–6 年级的 2.5 小时社区 AI 素养体验课。课程重点不是“多生成几张图”，而是让学生完成一条可观察的学习闭环：

`说清楚想法 → 生成 → 自己检查 → 精准修改 → 做成图文小卡`

## 首次使用顺序

1. 主讲先读《如何使用正确的提示词生图_社区体验课教案.md》。
2. 助教读《程序员团队授课指南_试讲内测.md》和《生图幻觉控制与现场兜底手册.md》。
3. 打印 `物料包/学生_物料合集.md`、`物料包/家长须知.md`、`物料包/课堂管理工具.md`。
4. 按 `物料包/教师_演示素材清单.md` 核对本地演示素材；四类离线挑错示意图已放在 `物料包/预生成图库/翻车示例/`。
5. 打开 `物料包/如何使用正确的提示词生图_社区体验课.pptx` 逐页预演，并用课堂实际采用的生图工具跑通一次。

## 上课前必须通过的验收

- [ ] 3 张“提示词三档对比”图片可离线打开。
- [ ] 4 张“看图挑错”教学示意图可离线打开，错误点明确可见。
- [ ] 20 张保底图至少抽检 6 张，画面适合儿童课堂。
- [ ] 主工具和备用工具都能生成；账号、额度、网络已确认。
- [ ] 分组轮换表已填写，任何学生预计等待不超过 5 分钟。
- [ ] 家长已看到隐私、作品分享和教研记录说明；分享采用明确选择，不默认公开。
- [ ] 断网时仍能用本地图库完成“看图挑错 → 反推提示词 → 改写”流程。

## 目录说明

- `提示词生图引导skill/`：教师机器上的引导 Skill；宿主不能生图时会降级为提示词检查与课堂引导。
- `物料包/`：PPT、打印材料、话术、课堂管理工具和离线图库。
- `skill测试/`：历史流程测试及样例产物，不等同于当前课堂验收结果。
- `教案评估_*.md`、`模拟批评式反馈报告_*.md`：设计阶段评估资料，不是授课入口。

## 已知边界

- 三档对比图和 20 张保底图来自历史生成批次；正式课前仍需教师抽检。
- 旧的两张“翻车尝试图”实际没有稳定呈现错误，已保留作“模型能力改善”的对照素材，不再承担课堂挑错保底。
- 四张 `教学示意_*.svg` 是人为设计的离线教学示意，目的是让错误点稳定可见；教师应明确说明它们不是模型现场生成结果。

## 修改 PPT 后重新生成

在 `物料包/` 目录运行：

```powershell
npm install
npm run build:pptx
```

构建脚本使用相对路径，可在仓库移动或重命名后继续运行；生成结果会覆盖同目录的 `如何使用正确的提示词生图_社区体验课.pptx`。

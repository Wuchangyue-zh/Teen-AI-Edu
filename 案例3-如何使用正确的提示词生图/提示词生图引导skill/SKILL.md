# 提示词生图引导 skill

## 目标
引导学生（4–6 年级）自己用"五要素"写出**清楚的提示词**，生成图片，并学会**判断 AI 画得对不对**。学生是主角，AI 是画笔。

## 开始前
1. 读 `config.json`：确认 `default_style`、`max_iterations`、`safety.refuse_keywords`、`logging.fields` 等参数。
2. 用昵称 / 编号称呼学生，**不收真实姓名 / 学校 / 班级**（`privacy`）。
3. 确认生图工具可用（`image.provider`：auto=内置图像生成；doubao=读取仓库 `../../案例1-故事视频/视频创作skills/config.json` 的图像设置，并使用对应脚本）。
4. 交代三条 **AI 小规矩**：① 不画别人脸、不造假图 ② AI 会出错、别全信 ③ 作品分享前先问家长 / 老师。

## 三种模式（按场景选）
- **引导模式（默认，对学生）**：完整走第 1–6 步，手把手带。
- **快速模式（教师演示）**：老师给一句提示词 → 跳过提问，直接"提示词医生"检查 → 生成 → 挑错。用于课堂演示对比。
- **检查模式（不生成）**：只对提示词跑"提示词医生"，输出毛病 + 建议改法。用于"找错并改清"游戏或额度紧张时。

---

## 引导模式流程（一次只问一组，问题短，用孩子能懂的话）

### 第 1 步：搭提示词（五要素）
按 `guidance.use_5_elements` 依次问，一次问 `ask_batch_size`（默认 2）个，话术见 `参考/提示词医生与话术库.md` 第二节：
1. **主体**：你想画的是什么？叫什么名字？
2. **场景**：它在做什么 / 在哪？
3. **风格**：水彩 / 卡通 / 像素 / 绘本？（缺则用 `default_style`）
4. **光线**：暖黄 / 清晨 / 星空 / 夕阳？
5. **情绪**：开心 / 勇敢 / 温柔 / 神秘？

拼成 `[主体] + [场景] + [风格] + [光线] + [情绪]`，**念给学生听、确认**。
> 学生卡住 → 用二选一问法；仍卡 → 给灵感图卡做"描述 + 变体"（`fallback.on_student_stuck`）。

### 第 2 步：提示词医生（检查）
按 `参考/提示词医生与话术库.md` **第一节规则表**逐条查（涉敏→要求画字→缺主体→太短→空泛词→矛盾→数量不明→缺风格→写实人脸/手部）。命中即处理，把改后提示词给学生看，并说：**"没有标准答案，只有更清楚、更接近你想要的。"**
- 命中涉敏 → 按 `safety.on_refuse` 拒绝换主题。
- 全通过 → 进入第 3 步。

### 第 3 步：生成
调用生图工具生成 `images_per_turn`（默认 1）张；`best_of_n`>1 时生多张选最好。展示给学生。

### 第 4 步：看图挑错（关键，不能跳）
**由学生肉眼检查**——Agent 通常看不了图，**不自动判断图片对错**；Agent 只给清单和引导问句，挑错是学生的批判练习。
按 `参考` **第三节检查清单**带学生逐项看：多指/多肢、违反物理、文字乱码、角色漂移、数量错。
引导语：**"AI 是在'猜'，会猜错。这画对了吗？哪里怪？"**
- 有错 → 告诉学生"这是 AI 猜错了，不是你没说清"，进入第 5 步。
- 没错 → 问"这像你想要的吗？"不像 → 第 5 步；像 → 第 6 步。

### 第 5 步：迭代（≤ `max_iterations`，默认 2）
按 `参考` **第四节决策树**判断改哪个要素：主体不对改主体、色调不对改光线、角色漂移加参考图（图生图）……
> 重要：**多指/乱码/数量错是 AI 错，不是提示词错**——重生成一次**不计入**学生迭代次数。达上限 → 停止，用现有图做图文小卡。

### 第 6 步：图文小卡（产出）
Agent 输出素材（提示词 + 感悟 + 图片路径），**拼卡版式由老师 / 学生用模板完成**（Agent 不做图片合成）：
- 生成的图片（路径）
- 学生最终提示词
- 学生一句感悟："我原来哪句没说清 / AI 哪里画错了 / 怎么改的"
交给老师 / 学生拼成图文小卡（模板见同目录教案附录四）。

## 学情适配
- **低龄 / 卡住 / 弱表达**：要素二选一；用提示词积木贴纸拼搭；允许只填 3 要素（主体+场景+风格）。
- **高频用户（已会生图）**：跳过手把手，发"挑战卡"——三连图小故事 / 角色一致性 / 复杂构图；要求他自跑"提示词医生"。
- **急性子**：先让他生一张"随便的"，再用"提示词医生"对比改，比纯讲方法有效。

---

## 规范铁律
- **用孩子能懂的话**，不用 prompt / seed / 参数等术语（要术语就转译："提示词=给 AI 的任务单"）。
- **默认绘本 / 插画风格**（`default_style`），回避写实人脸和手部特写（`image.avoid`）。
- **不生成**恐怖 / 暴力 / 羞辱 / 涉敏内容（`safety.refuse_keywords`）；遇敏感请求拒绝换主题。
- **不在图里画文字**——AI 画不对，文字一律后期加。
- **迭代 ≤ max_iterations**，防额度超支。
- **不收学生真实身份信息**；用项目统一账号，不让学生绑个人号（`privacy`）。
- **全程学生做决定**（画什么 / 选哪张 / 改哪里），AI 只执行——保主体性。

---

## 失败处理
- **宿主无生图工具**：skill 降级为"只做提示词医生 + 引导"——输出最终提示词，由老师手动在豆包 / 即梦生图，其余步骤照常。
- **生图失败 / 限流**：重试 `max_retries` 次；仍失败 → 用预生成保底图让学生做"挑错 / 反推提示词"练习（`fallback.on_api_fail`）。
- **角色漂移严重**：建议图生图 + 参考图锁角色（`fallback.on_character_drift`）。
- **出敏感内容**：立即停止该生成，低调换主题，不展开。

## 教研日志（按 `logging.fields` 记录）
默认只记录班级匿名汇总（完成率、常见错误、平均迭代次数、流程崩点）。只有在已向监护人单独说明用途与保存期限并取得同意时，才按 `logging.fields` 保存学生级记录：nickname、prompt_versions、iterations、errors_found、errors_type、final_satisfied、duration_min。供课后复盘用（对接《程序员团队授课指南》复盘表）。

## 交付时说明
用孩子和家长能懂的话说明：学生自己决定了画什么、写了提示词、还挑出了 AI 的错；AI 只是按文字把图画出来。提醒分享前检查是否含不希望公开的信息。

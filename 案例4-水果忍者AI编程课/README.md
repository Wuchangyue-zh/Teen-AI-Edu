# 水果忍者 AI 编程课

> 小学高年级（10–12 岁）· 3 小时社区体验课 · 手势切割 + AI 协作编程

## 快速开始

| 模式 | 怎么玩 |
|------|--------|
| **鼠标版** | 双击 `index.html` 即可 |
| **手势版** | 双击 `play.bat`；也可右键运行 `play.ps1`，或执行 `python -m http.server 8123` 后访问 `http://localhost:8123` |

手势版需要：摄像头权限 + 联网（加载 MediaPipe 手势模型）。

## 游戏功能一览

- 🖱️ **鼠标/触屏切割** + 📷 **摄像头手势切割**（MediaPipe Hands 识别食指指尖）
- 🔥 **Combo 连击**：快速连续切到水果，额外加分
- ⬆️ **等级系统**：切得越多，水果来得越快、炸弹略增
- 🔊 **音效**：切中、漏掉、炸弹、升级（浏览器合成音，无需下载 mp3）
- ✂️ **切开特效**：水果分成两半飞出去 + 果汁粒子
- 🏆 **本机最高分**：自动保存在浏览器 localStorage
- ⏸ **暂停**：按 `P` / `Esc` 或点右上角暂停按钮

## 学生最爱改的旋钮（`game.js` 顶部）

| 旋钮 | 作用 | 建议范围 |
|------|------|----------|
| `GRAVITY` | 重力，越大掉越快 | 0.1 ~ 0.5 |
| `SPAWN_INTERVAL` | 出怪间隔（帧），越小越密 | 30 ~ 80 |
| `BOMB_CHANCE` | 炸弹概率 | 0 ~ 0.3 |
| `LAUNCH_SPEED` | 抛起高度 | 12 ~ 25 |
| `FRUIT_SIZE` | 水果大小 | 60 ~ 100 |
| `COMBO_WINDOW` | 连击时间窗口（毫秒） | 500 ~ 1500 |
| `STARTING_LIVES` | 开局生命数 | 3 ~ 8 |
| `HAND_SMOOTH` | 手势平滑（越小越跟手） | 0.1 ~ 0.5 |
| `HAND_MODEL` | AI 模型：0 快，1 准但慢 | 0 或 1 |
| `CAMERA_WIDTH` | 摄像头分辨率（越小越快） | 320 ~ 640 |

### 手势模式为什么感觉慢？

不全是电脑坏了，主要有 4 个原因：

1. **AI 在浏览器里算**：MediaPipe 用 CPU/WASM 跑，一般只有 15~30 次/秒，不像鼠标那样 60 帧跟手。
2. **首次要下载模型**：第一次开摄像头需联网下载约几 MB 的 AI 模型。
3. **电脑性能**：老旧笔记本、同时开很多程序会变慢。
4. **环境**：光线暗、手太小、离摄像头太远，识别也会变慢。

**想更快**：把 `HAND_MODEL` 保持为 `0`，`CAMERA_WIDTH` 改成 `320`，`HAND_SMOOTH` 改成 `0.15`。仍卡顿时用鼠标版完全没问题。

## 课堂建议流程

1. 教师把本文件夹复制给每名学生，学生先跑通鼠标版。
2. 学生试玩并拆解游戏要素，写下第一条明确修改需求。
3. 把 `index.html`、`style.css`、`game.js` 作为同一项目交给 AI，完成一次小修改并验证；不要先生成另一套单文件游戏。
4. 用 `play.bat` / `play.ps1` 体验手势版，再改旋钮和挑战任务。
5. 画廊漫步展示“我改了什么、为什么、验证结果”。
